Welcome to Luckey API documentation!

Introduction

This site is the specification for the Luckey API (LAPI). The Luckey API is part of the back end that describes a RESTful Web Service in the overall architecture. LAPI provides a way to consumers to manipulate the actual data source. Therefore, LAPI is the middleware software that expose the data to the outside world, in order to be consumed by the Luckey mobile application.

API Specs

API keywords and terminology
This terms are used throughout the API Spec section.
  • URI. Unique Resource Identifier is the resource id that provides an interactions to the API.
  • CRUD. Create, retrieve, update and delete are the common operations that are available to manipulate a data source.
  • Entity. An entity refers to a particular set of data of a particular domain. This represent the class of a particular set of data. E.g. All user related data is represented by the user entity. Even if the data is a collection of data, the data is of a particular domain. For that case, the set of data within the collection is related to a particular entity. Notice that not all data for a particular domain is an entity. E.g. tables that relates other tables are not entities.
  • Record. A set of data related of a particular entity. E.g. One user is one record.
  • Records. A collection of records. E.g. ten users are ten records.

API specifications

  • All URIâ€™s are be written in lowercase.
  • The API have a root access.
  • All entities have implement CRUD.
  • For each entity, the URI are namespaced. E.g. /root/persons.
  • All request to the API return a response.
  • The internet media type for representing data is JSON.
  • The API MUST implement standard HTTP verbs (methods).
  • For create implementation HTTP POST method SHOULD be used.
  • For retrieve implementation HTTP GET method SHOULD be used.
  • For update implementation HTTP PUT method SHOULD be used.

The API root

The LAPI have a root access for consumers. This not return any data to the client, but a welcome to the API. Root access be represented by the URI:
/luckey

Entity namespace

For each entity, the URI to perform a request to a particular functionality are namespaced:
luckey/business

JSON Struct

For every API response, the JSON struct will be the same. This is for purposes to do the developing phase or usage of the API more faster and simple.
